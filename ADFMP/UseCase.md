# Сценарии использования приложения

Введем следующие определения:

_Домашний экран приложения_ - экран, который видит пользователь при открытии приложения.

_Главный экран приложения_ - экран приложения, который содержит основную механику приложения 
(в данном случае, кнопку для кликанья).

_Навигационные кнопки_ - стандартные сенсорные кнопки для навигации, встроенные в телефон: "Меню", "Домой" и "Назад".

## Начать новую игру

<ins>Действующее лицо</ins>: пользователь.

<ins>Предусловие</ins>:

* Пользователь еще не начал игру и находится на домашнем экране приложения.

<ins>Основной сценарий</ins>:

1. Пользователь нажимает на кнопку "Play".
2. Пользователь попадает на экран с созданием новой игры.
3. Пользователь заполняет все необходимые поля:
    * Имя болезни - текстовое поле.
    * Сложность - выбор из списка.
4. Пользователь нажимает на кнопку "Play".

<ins>Альтернативный сценарий</ins>:

* Пользователь вводит уже существующее название болезни.
    * Пользователь получает уведомление с просьбой придумать другое название.
* Пользователь хочет отменить операцию после п. 1-3
    * Пользователь нажимает навигационную кнопку "Назад".
* Пользователь хочет отменить операцию после п. 4
    * См. сценарий использования "Сброс игры".

<ins>Результат</ins>:

1. Пользователь создает новую игру.
2. Пользователь попадает на главный экран приложения.

## Продолжить игру

<ins>Действующее лицо</ins>: пользователь.

<ins>Предусловие</ins>:

* Пользователь уже начал игру и находится на домашнем экране приложения или на экране меню.

<ins>Основной сценарий</ins>:

1. Пользователь нажимает на кнопку "Play".

<ins>Альтернативный сценарий</ins>:

* Пользователь хочет отменить операцию
    * Пользователь нажимает навигационную кнопку "Назад".

<ins>Результат</ins>:

1. Пользователь возобновляет игру.
2. Пользователь попадает на главный экран приложения.

## Сбросить игру

<ins>Действующее лицо</ins>: пользователь.

<ins>Предусловие</ins>:

* Пользователь уже начал игру и находится на главном экране приложения.

<ins>Основной сценарий</ins>:

1. Пользователь нажимает на кнопку "Пауза".
2. Пользователь попаадает на экран меню.
3. Пользователь нажимает кнопку "Сброс игры".
4. Пользователь видит всплывающее окно с подверждением операции.
5. Пользователь нажимает кнопку "Сброс".

<ins>Альтернативный сценарий</ins>:

* Пользователь хочет отменить операцию после п. 1-4
    * Пользователь нажимает кнопку "Отмена" во всплывающем окне или нажимает навигационную кнопку "Назад".

<ins>Результат</ins>:

1. Пользователь сбрасывает игру.
2. Пользователь попадает на добмашний экран приложения.

## Поставить игру на паузу

<ins>Действующее лицо</ins>: пользователь.

<ins>Предусловие</ins>:

* Пользователь уже начал игру и находится на главном экране приложения.

<ins>Основной сценарий</ins>:

1. Пользователь нажимает на кнопку "Пауза" или выходит из приложения.

<ins>Альтернативный сценарий</ins>:

* Пользователь хочет отменить операцию
    * Пользователь нажимает кнопку "Play" в экране меню или на домашнем экране.

<ins>Результат</ins>:

1. Пользователь ставит игру на паузу, сохраняя ее состояние.
2. Пользователь попадает на экран меню или выходит из приложения.

## Выйти из игры

<ins>Действующее лицо</ins>: пользователь.

<ins>Предусловие</ins>:

* Пользователь находится на любом экране приложения.

<ins>Основной сценарий</ins>:

1. Пользователь нажимает на навигационную кнопку "Домой".

<ins>Альтернативный сценарий</ins>:

* Пользователь хочет отменить операцию
    * Пользователь нажимает иконку приложения.

<ins>Результат</ins>:

1. Пользователь ставит игру на паузу, сохраняя ее состояние.
2. Пользователь выходит из приложения.
