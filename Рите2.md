Конечно. Я изучил список нормативных документов и, опираясь на заложенные в них принципы (такие как полнота спецификации, фокус на жизненном цикле, надежности, безопасности, удобстве сопровождения и формальных процедурах приемки), подготовил вторую часть спецификации.

| | BR-008 | Система должна быть разработана с учетом минимизации стоимости владения (TCO), включая затраты на поддержку, сопровождение и будущие обновления. |
| | BR-009 | Система должна повысить лояльность существующих клиентов компании (если применимо) на 15% за счет предоставления ценного дополнительного продукта. |
| | BR-010 | Система должна иметь четко определенный жизненный цикл, включая стадии разработки, внедрения, эксплуатации и снятия с поддержки, в соответствии с ГОСТ 34.601-90. |

| | UR-008 | Пользователь должен иметь возможность откатить изменения настроек к последнему сохраненному состоянию или к состоянию по умолчанию. |
| | UR-009 | Пользователь должен иметь возможность заблокировать текущие значения экспозиции (AE Lock) и фокуса (AF Lock) для перекомпоновки кадра. |
| | UR-010 | Пользователь должен иметь возможность использовать таймер спуска затвора с настраиваемыми интервалами (например, 2, 5, 10 секунд). |
| | UR-011 | Пользователь должен иметь возможность быстрого доступа к справочной информации или подсказкам по каждой ручной настройке прямо из интерфейса. |
| | UR-012 | Пользователь должен иметь возможность выбрать место для сохранения фотографий (внутренняя память или SD-карта, если доступно). |
| **Функциональные** | FR-008 | Система должна предоставлять функцию "Focus Peaking" (подсветка зон в фокусе) для облегчения ручной фокусировки. |
| | FR-009 | Система должна отображать предупреждение о пересвете ("зебра") на участках изображения, которые теряют детализацию в светах. |
| | FR-010 | Система должна позволять выполнять серийную съемку (burst mode) при удержании кнопки спуска затвора. |

| | FR-012 | Система должна предоставлять возможность кадрирования с различными соотношениями сторон (1:1, 4:3, 16:9, 3:2) в реальном времени. |

| | SR-008 | Система должна поддерживать облачную синхронизацию пользовательских пресетов между несколькими устройствами одного пользователя. |
| | SR-009 | Система должна безопасно обрабатывать сценарии нехватки памяти или дискового пространства, информируя пользователя и предотвращая потерю данных. |
| | SR-010 | Система должна использовать аппаратное ускорение графики для отрисовки интерфейса и видоискателя. |
| | SR-011 | Система должна корректно приостанавливать свою работу (уходить в фон) и возобновлять ее из последнего состояния в соответствии с жизненным циклом Android-приложений. |
| | SR-012 | Система должна шифровать данные пользователя (профиль, данные для синхронизации) при их передаче по сети. |
| **Бизнес-правила** | RL-007 | Внесение изменений в архитектуру приложения требует обязательного прохождения процедуры ревью и утверждения ведущим архитектором. |
| | RL-008 | Поддержка новых версий ОС Android должна быть реализована не позднее, чем через 3 месяца после их официального релиза. |
| | RL-009 | Любая новая функциональность должна сопровождаться набором автоматизированных тестов с покрытием кода не менее чем на 80%. |

| | RL-012 | Перед выпуском мажорного обновления должна быть выпущена бета-версия для ограниченной группы пользователей с целью сбора обратной связи. |

| | QA-008 | **(Ремонтопригодность)** Время, необходимое для анализа, исправления и развертывания исправления критической ошибки, не должно превышать 48 часов. |

| | QA-010 | **(Безопасность)** Система должна соответствовать требованиям OWASP Mobile Top 10 для защиты от известных уязвимостей мобильных приложений. |
| | QA-011 | **(Совместимость)** Приложение должно корректно работать на 95% самых популярных моделей Android-устройств на целевом рынке. |
| | QA-012 | **(Тестируемость)** Каждое требование в данной спецификации должно быть проверяемо посредством инспекции, анализа, демонстрации или автоматизированного теста (согласно ГОСТ 34.602-89). |

| | UI-008 | Система должна использовать тактильный отклик (вибрацию) для подтверждения важных действий (например, сохранение пресета, смена режима). |
| | UI-009 | Все иконки в интерфейсе должны быть интуитивно понятны без текстовых подписей, но должны отображать текстовую подсказку при долгом нажатии. |
