Все формулировки соответствуют критериям "корректного" требования: они атомарны, однозначны, проверяемы и используют конструкцию «система/приложение должно».

### Спецификация требований к приложению «МVx-камера»

| Тип требования | Уникальный идентификатор | Формулировка требования |
| :--- | :--- | :--- |
| **Бизнес-требования** | BR-001 | Система должна обеспечить привлечение и удержание целевой аудитории (фотографы-энтузиасты и профессионалы) для занятия ниши специализированных фото-приложений. |
| | BR-002 | Система должна поддерживать модель монетизации через платные премиум-функции или подписку для обеспечения финансовой выгоды. |
| | BR-003 | Система должна способствовать укреплению бренда компании как инноватора в сфере мобильной фотографии. |
| | BR-004 | Система должна стимулировать виральный маркетинг, предоставляя пользователям возможность делиться результатами съемки с упоминанием приложения. |
| | BR-005 | Система должна обеспечить стратегическое преимущество путем расширения экосистемы продуктов компании и повышения конкурентоспособности. |
| | BR-006 | Система должна предусматривать возможность интеграции с партнерскими сервисами (фотостоки, сервисы печати) для получения комиссионного дохода. |
| **Пользовательские** | UR-001 | Пользователь должен иметь возможность вручную управлять значением ISO для контроля светочувствительности сенсора камеры. |
| | UR-002 | Пользователь должен иметь возможность вручную управлять выдержкой для контроля времени экспозиции и создания эффектов движения. |
| | UR-003 | Пользователь должен иметь возможность вручную управлять точкой фокусировки для точного наведения на объект съемки. |
| | UR-004 | Пользователь должен иметь возможность создавать, сохранять и именовать собственные наборы настроек (пресеты) для быстрого применения в будущем. |
| | UR-005 | Пользователь должен иметь возможность быстро переключаться между сохраненными пресетами для адаптации к различным условиям съемки. |
| | UR-006 | Пользователь должен иметь возможность выбрать формат сохранения снимка из доступных вариантов (JPEG, RAW, HEIF). |
| **Функциональные** | FR-001 | Система должна предоставлять ползунок (слайдер) для ручной регулировки значения ISO в диапазоне, поддерживаемом аппаратным обеспечением устройства. |
| | FR-002 | Система должна предоставлять ползунок (слайдер) для ручной регулировки выдержки в диапазоне, поддерживаемом аппаратным обеспечением устройства. |
| | FR-003 | Система должна отображать на экране видоискателя сетку для помощи в композиции кадра (например, "правило третей"). |
| | FR-004 | Система должна позволять сохранять пользовательский пресет, который включает в себя текущие значения ISO, выдержки, баланса белого и фокуса. |
| | FR-005 | При выборе сохраненного пресета система должна автоматически применять все сохраненные в нем параметры к текущим настройкам камеры. |
| | FR-006 | Система должна предоставлять функционал для удаления ранее созданных пользовательских пресетов. |
| | FR-007 | Система должна отображать гистограмму в реальном времени для оценки экспозиции кадра. |
| **Системные** | SR-001 | Система должна использовать Camera2 API для обеспечения доступа к расширенным ручным настройкам камеры на устройствах под управлением Android. |
| | SR-002 | Система должна запрашивать у пользователя и получать все необходимые разрешения для доступа к камере, микрофону и хранилищу устройства при первом запуске. |
| | SR-003 | Система должна сохранять созданные пользователем пресеты в локальном хранилище приложения. |
| | SR-004 | Система должна корректно работать на устройствах под управлением ОС Android версии 7.0 (API level 24) и выше. |
| | SR-005 | Система должна поддерживать переключение между всеми доступными на устройстве физическими камерами (например, основной, широкоугольной, телеобъективом). |
| | SR-006 | Система должна адаптировать свой интерфейс для корректного отображения на экранах с различным разрешением и соотношением сторон. |
| **Бизнес-правила** | RL-001 | Приложение не должно содержать функционал для постобработки изображений (фильтры, ретушь), так как оно ориентировано исключительно на процесс съемки. |
| | RL-002 | Бесплатная версия приложения должна позволять создавать и хранить не более трех пользовательских пресетов. |
| | RL-003 | Сохранение фотографий в формате RAW должно быть доступно только в платной (премиум) версии приложения. |
| | RL-004 | Для использования облачной синхронизации пресетов пользователь должен пройти процедуру регистрации и авторизации. |
| | RL-005 | Приложение не должно отображать рекламу в платной (премиум) версии. |
| | RL-006 | Все снимки, сделанные в бесплатной версии, при экспорте в социальные сети должны по умолчанию содержать упоминание (водяной знак или текст) «Снято на МVx-камера». |
| **Атрибуты качества** | QA-001 | Время запуска приложения от момента касания иконки до полной готовности к съемке не должно превышать 2 секунды на устройствах среднего класса. |
| | QA-002 | Задержка между изменением параметра (например, ISO) и его визуальным применением на экране видоискателя не должна превышать 0.2 секунды. |
| | QA-003 | Интерфейс приложения должен работать с частотой обновления не менее 60 кадров в секунду (FPS) на всех поддерживаемых устройствах. |
| | QA-004 | Частота критических сбоев приложения (crash rate) не должна превышать 0.5% от общего числа сеансов использования. |
| | QA-005 | Приложение должно успешно сохранять все пользовательские данные (включая пресеты) при обновлении до новой версии. |
| | QA-006 | Среднее время, необходимое пользователю для полной ручной настройки основных параметров (ISO, выдержка, фокус), не должно превышать 10 секунд. |
| **Внешний интерфейс** | UI-001 | Система должна отображать все основные элементы управления (ISO, выдержка, баланс белого, фокус) на главном экране видоискателя без необходимости перехода в другие меню. |
| | UI-002 | Система должна обеспечивать доступ к меню выбора пресетов не более чем в два касания с главного экрана. |
| | UI-003 | Приложение должно соответствовать гайдлайнам Material Design, принятым для Android-разработки. |
| | UI-004 | Система должна предоставлять визуальную обратную связь при изменении любого параметра съемки (например, изменение яркости изображения при смене ISO). |
| | UI-005 | Система должна иметь "ночной режим" интерфейса (темную тему) для комфортного использования в условиях низкой освещенности. |
| | UI-006 | Система должна позволять пользователю скрыть/показать вспомогательные элементы интерфейса (например, сетку или гистограмму) одним касанием. |
| **Ограничения** | CN-001 | Система должна информировать пользователя, если выбранная ручная настройка (например, определенное значение выдержки) не поддерживается аппаратным обеспечением текущего устройства. |
| | CN-002 | Приложение должно быть разработано исключительно для операционной системы Android. |
| | CN-003 | Минимальный объем оперативной памяти для стабильной работы приложения должен составлять 6 гигабайт. |
| | CN-004 | Приложение должно использовать только встроенные в ОС системные шрифты. |
| | CN-005 | Разработка должна вестись с использованием языка программирования Kotlin. |
| | CN-006 | Система не должна требовать постоянного подключения к сети Интернет для своей основной работы (ручной съемки). |
